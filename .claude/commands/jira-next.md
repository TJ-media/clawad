담당자가 김태정인 CLAW 이슈 중 작업 가능한 것을 최대 3개 추천하고, 선택 즉시 자동 개발 파이프라인을 시작한다.

---

## 1. 이슈 추천

`Agent` 도구로 `.claude/agents/jira-recommender.md`에 정의된 `jira-recommender` 에이전트를 호출한다.

에이전트가 `RECOMMEND_EMPTY`를 반환하면:
- "작업 가능한 이슈가 없습니다. Jira 보드를 확인해주세요."를 출력하고 종료한다.

---

## 2. 결과 출력

에이전트의 결과를 아래 형식으로 변환해 출력한다.

---

## 🎯 작업 가능한 이슈 (상위 3개)

| 순위 | 이슈 키 | 제목 | 의존성 |
|------|---------|------|--------|
| 1 | CLAW-XX | ... | ✅ 없음 / ⚠️ 선행 이슈 미완 |
| 2 | CLAW-XX | ... | ... |
| 3 | CLAW-XX | ... | ... |

### 각 이슈 요약

**1. CLAW-XX — {제목}**
- 구현 범위: {한 줄 요약}
- 추천 브랜치명: `feat/claw-XX`

(2, 3번도 동일 형식)

---

## 3. 선택 및 파이프라인 시작

출력 후 사용자에게 묻는다:

> 작업할 이슈 번호(1 / 2 / 3)나 이슈 키(CLAW-XX)를 입력해주세요.
> 선택하면 자동 개발 파이프라인을 바로 시작합니다.

사용자가 응답하면:
- 번호(1 / 2 / 3) 입력 시 → 해당 순위의 이슈 키를 `SELECTED_KEY`로 설정한다.
- 이슈 키(CLAW-XX) 직접 입력 시 → 그대로 `SELECTED_KEY`로 설정한다.
- 그 외 입력 시 → "올바른 번호나 이슈 키를 입력해주세요."를 출력하고 다시 묻는다.

`SELECTED_KEY`가 결정되면 `auto-dev.md`의 전체 파이프라인을 `$ARGUMENTS = SELECTED_KEY`로 실행한다.
