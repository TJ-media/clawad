인자로 받은 내용을 바탕으로 Jira 일감 등록 계획을 수립하고, 사용자 승인 후 CLAW 프로젝트에 이슈를 생성한다.

인자가 없으면 "등록할 일감 내용을 인자로 전달하세요. 예: /create-jira 광고주 셀프서브 포털 구현"을 출력하고 종료한다.

---

## 1단계: 이슈 유형 판단

인자($ARGUMENTS)와 대화 맥락을 바탕으로 아래 유형 중 하나를 결정한다.

| 유형 | issueTypeName | 판단 기준 |
|------|--------------|-----------|
| Feature | `작업` (프로젝트에 Feature 유형이 있으면 `Feature`) | 신규 기능, 화면, API 구현 |
| Bug | `버그` | 오류, 에러, 깨진 동작, 수정 |
| Task | `작업` | 리팩터링, 설정, 문서, 환경 구성 |

우선순위도 판단한다: `Highest` / `High` / `Medium` / `Low`
- 정산·집계 무결성·데이터 손실 관련 → High 이상
- 일반 기능 → Medium
- 개선·리팩터링 → Low

### CLIENT/SERVER 분리 원칙 (필수)

하나의 요청에 클라이언트와 서버 작업이 함께 필요하면 **반드시 `[CLIENT]`와 `[SERVER]` 두 개의 이슈로 나눠서 작성한다.**

- 분리 기준
  - `[CLIENT]`: 상태줄 렌더링, 로컬 원장, sync, VS Code 익스텐션 등 개발자 머신에서 도는 코드
  - `[SERVER]`: 광고 서빙/경매, 노출 수집, 정산, 광고주 포털 등 서버 코드
  - `[INFRA]`: 레포 설정, CI, 배포, 문서 등
- 각 이슈는 상대 이슈에 의존하는 부분을 `[연관 일감]` 항목에 명시한다.
- 작업이 한쪽으로 끝나면 분리하지 않고 한 개로 작성한다.
- 여러 개선사항이 한 번에 들어오면 사안별로 위 규칙을 각각 적용한다.

## 2단계: 템플릿 기반 이슈 초안 작성

### Feature / Task 템플릿

```
[설명]
* (한 문장으로 기능/작업 설명)

[요구사항]
* (구현해야 할 사항을 bullet로)
* (제약 조건)

[API]
* (Method) (URL) — 없으면 "해당 없음"

[Request]
* (요청 필드 및 타입) — 없으면 "해당 없음"

[Response]
* (응답 필드 및 상태값) — 없으면 "해당 없음"

[데이터]
* (관련 파일/저장 구조 — ledger, ads.json, DB 등) — 없으면 "해당 없음"

[예외]
* (예외 케이스 → 처리 방식)
```

### Bug 템플릿

```
[설명]
* (버그 한 문장 설명)

[재현 조건]
* (어떤 상황에서 발생하는지)
* (재현 순서)

[현재 동작]
* (잘못된 동작)

[기대 동작]
* (올바른 동작)

[원인 분석]
* (파악된 원인 — 모르면 "미파악")

[수정 범위]
* (수정이 필요한 파일/함수/로직)

[예외]
* (관련 엣지 케이스)
```

## 3단계: 초안 출력 및 사용자 확인

### 제목 형식 규칙

```
[CLIENT] 한국어 제목    ← 클라이언트 작업
[SERVER] 한국어 제목    ← 서버 작업
[INFRA] 한국어 제목     ← 인프라/설정/문서
```

예시:
- `[CLIENT] 상태줄 광고 로테이션 주기 서버 제어 지원`
- `[SERVER] 광고주 셀프서브 등록 API 구현`

명확하지 않으면 `[CLIENT]`를 기본으로 사용한다.

---

아래 형식으로 출력한 뒤 **반드시 사용자에게 확인을 받는다.**

```
📋 Jira 등록 계획

제목: (작성한 summary)
유형: (Feature / Bug / Task)
우선순위: (Highest / High / Medium / Low)

설명:
(작성한 description 전문)

---
Jira에 등록할까요?
```

사용자가 수정을 요청하면 반영해 다시 확인받는다.
사용자가 "응", "ㅇㅇ", "등록", "yes", "y" 등 긍정 응답을 하면 4단계로 진행한다.
(단, 사용자가 이미 "일감 등록하면서 진행해"처럼 포괄 승인한 세션에서는 확인 없이 진행할 수 있다.)

## 4단계: Jira 이슈 생성

Atlassian MCP의 이슈 생성 도구(createJiraIssue 류)로 생성한다.

- cloudId: `d4081ac1-010a-45f5-8241-d9d67209e21b`
- projectKey: `CLAW`
- issueTypeName: (3단계에서 결정한 유형 — 프로젝트에 존재하는 유형명 사용)
- summary / description: (3단계 확정본)
- contentFormat: `markdown`
- additional_fields:
  - priority: `{ "name": "(결정한 우선순위)" }` (프로젝트가 priority 필드를 지원할 때만)
  - assignee_account_id: `712020:5c7166ce-43b2-42c3-9acf-8c0a495dbaf4`

생성 완료 후:

```
✅ Jira 이슈가 등록됐습니다.
CLAW-XXX: (summary)
https://whatsuphouse.atlassian.net/browse/CLAW-XXX
```
