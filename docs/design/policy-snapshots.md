# 결정 시점 정책 스냅샷 (CLAW-44)

광고 노출의 유효성, 과금, 리워드는 이벤트 업로드나 배치 실행 시점의 최신 설정이 아니라 광고 결정 시점의 정책을 따른다. 그래야 정책 또는 캠페인 설정이 바뀌어도 이미 발생한 거래를 같은 입력으로 재현할 수 있다.

## 저장과 검증

- 서버는 광고 결정 시 `policyVersion`, `rewardPolicyId`, 과금·리워드 자격, 노출 단가, 리워드율, viewability, 동시 노출 허용 오차, 계정·캠페인·광고주 상한을 하나의 스냅샷으로 만든다.
- 스냅샷 본문과 `policySnapshotId`를 serveToken에 서명한다. 클라이언트는 토큰을 불투명 값으로 보관·제출할 뿐 정책값을 만들거나 변경하지 않는다.
- 같은 내용은 SHA-256 내용 해시로 하나의 `decision_policy_snapshots` 레코드를 참조한다. 이 테이블은 UPDATE·DELETE를 금지하는 append-only 원본이다.
- 이벤트 수집 시 토큰 서명뿐 아니라 스냅샷 ID·내용 해시·정책 버전이 저장된 원본과 일치하는지 확인한다.
- `impression_events`, `billing_ledger`, `reward_ledger`는 스냅샷 ID와 감사에 필요한 단가·리워드율·자격 정보를 보존한다. 환불, 회수, 동시 노출 재투영도 원래 스냅샷을 승계한다.

## 발급 뒤 변경의 처리

- 정책 파일, 캠페인 단가, 리워드 정책 또는 상한이 바뀌어도 이미 발급된 유효 토큰은 서명된 스냅샷으로 판정·과금·리워드한다.
- 캠페인 일시중지·종료는 신규 토큰 발급을 막는다. 이미 발급된 토큰은 짧은 TTL 동안 기존 스냅샷을 따른다.
- 즉시 중단이 필요한 보안·법적 상황은 캠페인·사용자·기기 kill switch가 우선하며, 해당 이벤트는 `KILLED`로 거절한다.
- 토큰 만료 후에는 새 광고 결정과 새 정책 스냅샷이 필요하다.

이 규칙은 과거 거래를 소급 변경하지 않으면서도 긴급 차단 수단을 유지한다.
