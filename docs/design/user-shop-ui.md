# 사용자 리워드 샵 웹 설계 (user-web)

> 상태: 설계. 구현 이슈는 Jira 신규 등록(CLAW-36 예정). CLAW-26 교환 API를 소비하는 사용자 화면이다.
> 원칙: 알파는 **최소 노력** — 프레임워크 없는 정적 페이지 1장. 기존 기프티콘 샵의 표준 레이아웃 관습을 따르되
> 타사 코드·이미지·로고 에셋은 복제하지 않는다(브랜드명은 텍스트만).

## 1. 왜 필요한가

CLAW-26은 교환 API(`GET /v1/rewards/products`, `POST /v1/rewards/redeem`, `GET /v1/rewards/redemptions`)까지만
구현했다. 사용자가 포인트를 확인하고 상품을 골라 교환할 화면이 없으면 알파 테스터는 curl을 쳐야 한다.

## 2. 화면 구성 (기프티콘 샵 표준 패턴)

한 페이지(SPA 아님, 섹션 전환만):

```
┌────────────────────────────────────────────────┐
│ 클로애드 리워드 샵      [내 확정 포인트 3,250P] [교환 내역] │  ← 헤더
├────────────────────────────────────────────────┤
│ (전체) (구독권) (카페) (편의점) (상품권)              │  ← 카테고리 필터 칩
├────────────────────────────────────────────────┤
│ 개발자 픽 · 목표 상품                                │
│ ┌─────────────────┐ ┌─────────────────┐          │
│ │ Claude Pro 1개월  │ │ ChatGPT Plus 1개월│          │  ← 최상단 구독권 (앵커)
│ │ 30,000P [부족]   │ │ 30,000P [부족]    │          │
│ └─────────────────┘ └─────────────────┘          │
│ 지금 교환 가능                                      │
│ ┌────────┐ ┌────────┐ ┌────────┐                 │
│ │편의점3천 │ │아메리카노│ │문화상품권│                 │  ← 도달 가능 상품 그리드
│ │3,000P교환│ │4,500P   │ │5,000P  │                 │
│ └────────┘ └────────┘ └────────┘                 │
│ 고지: 비구매·비양도, 비제휴, 수동 발송 1~2일           │  ← 푸터 고지
└────────────────────────────────────────────────┘
```

- **상품 카드**: 브랜드명(텍스트) · 상품명 · 포인트가 · [교환] 버튼. 잔액 부족이면 버튼 비활성 + "부족".
- **교환 버튼** → 확인 모달("3,000P가 차감됩니다") → `POST /v1/rewards/redeem` → 성공 시 "신청 완료,
  운영자가 1~2일 내 발송" 안내.
- **교환 내역**: `GET /v1/rewards/redemptions` — 상태 배지(신청됨/발송 완료/실패/취소).
- **로그인**: 소셜 전용(Google·Kakao·Naver) 버튼 → `POST /v1/auth/social/:provider/start`로 authorization URL을
  받아 이동 → 콜백 복귀 시 URL fragment의 1회성 handoff code를 `POST /v1/auth/social/exchange`로 교환(CLAW-37).
  최초 로그인은 서비스 약관·개인정보 필수 동의 모달을 거친다. accessToken은 메모리 보관(브라우저 영구 저장소 금지,
  탭 닫으면 재로그인 — 알파 수용). 이메일/비밀번호·GitHub 공개 로그인은 제공하지 않는다.

## 3. 상품 구성 (초기 카탈로그 제안)

| 구역 | 상품 | 포인트(제안) | 비고 |
|---|---|---|---|
| 최상단 앵커 | Claude Pro 1개월 이용권 | 30,000P | 목표 상품. 일일 상한 150P 기준 200일 — 도달용 아닌 동기부여용 |
| 최상단 앵커 | ChatGPT Plus 1개월 이용권 | 30,000P | 동일 |
| 교환 가능 | 편의점 금액권 (GS25·CU·세븐일레븐) | 3,000·5,000P | 액면가 |
| 교환 가능 | 카페 아메리카노 (스타벅스·투썸·메가·컴포즈·빽다방·바나프레소) | 1,500~4,700P | 실제 판매가 |
| 교환 가능 | 문화상품권 (컬쳐랜드) | 5,000·10,000P | |

- 포인트가는 전부 **products 테이블 값**이다(코드 하드코딩 없음, 운영자 API로 등록). 시드: `scripts/seed-catalog.mjs`.
- **최소 교환액은 1,500P**(CLAW-36에서 3,000→1,500 하향). 저가 카페 아메리카노(메가 2,500·컴포즈 1,500 등)를 실제 가격으로 넣기 위함. 첫 교환이 더 빨라져(≈10일) "빠른 보상" 방향과 일치.
- 실제 가격: 스타벅스 4,700·메가 2,500은 리서치 확인. 투썸·컴포즈·빽다방·바나프레소는 근사 — **알파 오픈 전 운영자가 실제 기프티콘 가격으로 최종 확정**한다.
- **구독권 운영 확인 필요**: Claude Pro·ChatGPT Plus는 공식 "선물하기"가 없을 수 있다. 알파 수동 지급
  방식(대납·코드 발급 등)을 운영자가 확정한 뒤 카탈로그에 올린다. 확정 전엔 미등록.
- 교환 신청이 뜨면 운영자가 대기 큐(`GET /internal/v1/redemptions/pending`)에서 보고 수동 발송.

## 4. 표기·법적 고지 (필수)

- **비제휴 고지**: "상품명·상표는 각 사의 소유이며 클로애드는 Anthropic·OpenAI 등과 제휴 관계가 없습니다."
  (rules §7 — Claude 공식 서비스 오인 금지. 구독권을 상품으로 팔더라도 공식 제휴로 보이면 안 된다.)
- **리워드 성격**: "비구매형·비양도형, 지정 상품 교환 전용" (CLAW-14·16).
- **수동 발송 안내**: "알파 기간에는 운영자가 수동 발송합니다(1~2일 소요)."
- 브랜드 로고 이미지는 사용하지 않는다(권리 미확보). 텍스트 브랜드명만.

## 5. 기술 방식 (최소 노력)

- `apps/user-web/index.html` **단일 파일** — 프레임워크·빌드 없음, 바닐라 HTML+CSS+JS.
- API 호출은 `fetch`, 서버는 기존 NestJS(`localhost:3111`)에 CORS 허용 추가만.
- 배포: 알파는 로컬/정적 호스팅 아무거나. 인증 토큰은 메모리 보관.
- 카테고리 필터·잔액 비교·버튼 상태는 전부 클라이언트 JS(서버 변경 없음, products에 category 필드만 추가 검토).
- 목업: 대화 중 위젯으로 확정한 레이아웃(포인트 배지 헤더, 필터 칩, 앵커 2카드 + 그리드) 기준.

## 6. 하지 않는 것 (알파)

- 별도 회원가입 화면(가입은 소셜 최초 로그인 + 동의로 처리 — CLAW-37)
- 결제·충전(리워드는 비구매형 — 구현 금지 항목), 장바구니, 찜
- 브랜드 로고·상품 이미지, 반응형 완벽 대응(데스크톱 우선)
- 자동 발송(벤더 API = CLAW-35, P2)
