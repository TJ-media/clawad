# 광고 심사 기준·금지 업종 정책 (CLAW-20)

> 상태: 확정 (v1). **광고는 등록 즉시 노출되지 않는다.** 운영자 콘솔(CLAW-25) 심사 승인 후에만 활성화한다.

## 1. 금지 업종 (등록 불가)

불법 서비스, 도박, 성인물, 고수익 투자 유도, 가상자산 과장 광고, 개인정보 판매, 악성 확장프로그램, 취업 사기, 자격증·합격 보장, 대출, 의약품, 경쟁사·타인 상표 사칭.

- 코드화: `gambling`, `adult`, `hyip`, `crypto_hype`, `pii_sale`, `malware`, `job_scam`, `cert_guarantee`, `loan`, `pharma`, `trademark_infringe`, `illegal`.

## 2. 문구 심사 기준

- Claude Code/IDE의 **공식 메시지·오류 메시지로 오인될 문구 금지**. 예: "업데이트 필요: 여기를 클릭", "보안 경고: 인증 만료", "Claude 추천 개발자 도구"(실제 추천 아닌 경우).
- Anthropic·Claude와 제휴/후원/추천 관계를 시사하는 표현 금지 (비제휴 원칙).
- 과장·허위·비교 비방, 낚시성 클릭 유도 금지.
- 길이 3~60자, 링크·이모지 남용 제한.

## 3. [광고] 표기 강제

- 상태줄 광고에는 시스템이 `[광고]`를 자동 부착한다. 광고주가 제거·변경할 수 없으며 심사 대상이 아니다(항상 부착).
- 예시: `[광고] 원티드 · 백엔드 개발자 채용 중 │ 확정 320P`

## 4. 심사 워크플로

```
등록 → 대기(pending) → 운영자 검토
      ├─ 승인(approved) → 활성(active) → 노출
      └─ 반려(rejected, 사유 코드) → 광고주 통지 + 예산 환불(광고주 약관)
```
- 승인 후 크리에이티브(문구·URL·브랜드) 변경 시 **재심사 필수**(active → pending).
- 반려 사유 코드: `prohibited_industry`, `impersonation`, `misleading`, `bad_landing`, `trademark`, `format`.

## 5. 랜딩 URL 검증

- 최종 도착 URL과 리다이렉트 체인 확인, 악성/피싱 사이트 차단.
- HTTPS 권장, 도메인 소유·브랜드 일치 확인.
- P2에서 자동 검증(CLAW-31)로 1차 필터 후 수동 최종 승인.

## 5.5 캠페인 유형별 심사 (PAID / HOUSE / TEST)

- **PAID**: 정식 심사 대상. 등록 → 대기 → 승인/반려 → 활성. 반려 시 예산 환불(광고주 약관).
- **HOUSE**(자사 광고): 과금은 없지만 **동일한 표시·문구 원칙을 그대로 적용**한다. 하우스 광고도 `[광고]` 표기를 붙이고, Claude Code 공식 메시지·오류로 오인되지 않아야 하며, Anthropic 제휴를 시사하지 않는다. 내부 승인 절차로 관리한다.
- **TEST**: 테스트 인벤토리로만 노출하며 실제 사용자 리워드·광고주 매출과 분리한다. 운영 리포트에 실매출·실리워드로 섞이지 않도록 한다.

> 표시 원칙은 유료·하우스·테스트 모두 동일하다. 과금·리워드 자격만 캠페인 유형으로 다르게 강제한다(CLAW-6 §캠페인, `server/lib/campaign.js`).

## 6. 단계별 적용

- MVP(P1): 운영자 콘솔에서 **수동 심사**(CLAW-25). 광고주는 이메일·계약으로 소재 제출.
- 확장(P2): 자동 1차 분류 + 수동 최종 승인(CLAW-31), 셀프서브 포털(CLAW-4)에 연동.

## 7. 구현 연계
- 운영자 심사 UI·상태 전이: CLAW-25 / 자동 심사: CLAW-31 / 환불 규정: CLAW-16.
