# 무효 트래픽·리워드 회수·이의제기 정책 (CLAW-19)

> 상태: 확정 (v1). 검증 파이프라인(CLAW-6)·이용약관(CLAW-16)·QA(CLAW-29)가 이 정책을 근거로 삼는다.

## 1. 무효 트래픽(IVT) 정의

아래는 유효 노출로 인정하지 않는다:

- 헤드리스/자동화 스크립트로 생성한 노출 (사용자 부재)
- 24시간 연속 등 비현실적으로 긴 연속 노출
- 실제 개발 세션이 없는 상태의 노출 (Claude Code 미실행)
- 물리적으로 불가능한 이벤트 간격(5초 미만 연속 완결 등)
- 동일 기기 지문의 다계정 운영
- 백그라운드/유휴 세션에서의 노출

## 2. 판정 2단계

| 단계 | 시점 | 처리 |
|---|---|---|
| 실시간 필터 | 이벤트 수집 시(CLAW-6) | 명백한 위반은 즉시 rejected + 사유 코드, 미적립 |
| 사후 배치 검수 | 리워드 확정 전 | 패턴 분석(연속성·세션·지문)으로 accrue_pending → 확정 또는 회수 판정 |

## 3. 사유 코드 체계

`bad_token`, `expired`, `over_cap`, `bad_interval`, `no_session`, `headless_suspect`, `multi_account`, `background`, `killed`, `manual_review`.
- 모든 거절·회수는 `ad_event_ledger.reject_reason` / `reward_ledger.reason`에 코드로 기록(분쟁 대응).

## 4. 회수 규칙

- 사후 부정 판정 시: **광고주 크레딧 복원**(billing.ivt_refund) + **사용자 리워드 회수**(reward.claw_back). 원장에 근거 노출·사유 기록.
- 이미 교환·발송된 리워드는 회수 불가 → 다음 확정분에서 차감하거나, 악의적 대량 부정은 계정 정지 후 미지급 처리.

## 5. 지급 보류

- 부정 의심(스코어 임계 초과, 다계정 연루 등) 사용자의 교환 신청은 **보류** 상태로 접수(202).
- 해제 조건: 수동 검수 통과 또는 이의제기 인용. 보류 기한 내 미소명 시 반려.

## 6. 이의제기 절차

- 접수 채널: [고객센터/이메일]. 
- 처리 기한: 접수 후 [영업일 N일] 내 1차 통지.
- 판정 근거 공개 범위: 어뷰징 탐지 우회를 막기 위해 상세 룰은 비공개, 다만 사유 코드와 개괄 사유는 안내.
- 인용 시 구제: 오판정으로 확인되면 리워드 재적립(reward.admin_adjust)·보류 해제.
- 재발 기준: 반복 위반 [N회] 시 계정 이용정지, 중대·고의는 즉시 해지(이용약관 근거).

## 7. 광고주 측 처리

- IVT로 판정된 노출은 과금하지 않으며, 이미 과금분은 크레딧 환급.
- 캠페인 리포트는 사후 검수 완료 후 확정(잠정치 → 확정치, CLAW-16).

## 8. 구현 연계
- 실시간·배치 판정: CLAW-6 / 시나리오 테스트: CLAW-29 / 지급 보류: CLAW-26 / 계정 조치·감사: CLAW-27.
