# 수익·정산·리워드 정책 (CLAW-12)

> 상태: 초안 확정 (v1). 배분율·상한·단가 변경은 이 문서 개정으로만 한다. **코드에 수치 하드코딩 금지** — 서버 정책 테이블로 관리한다.

## 1. 사업 정의

**클로애드는 광고주가 구매한 개발자 대상 광고 인벤토리를 Claude Code/IDE 사용자에게 제공하고, 검증된 광고 매출의 일부를 비현금성 리워드로 배분하는 광고 매체 플랫폼이다.**

클로애드는 지급수단(포인트) 발행자가 아니다. 리워드는 비구매형·비양도형이며, 클로애드가 지정한 상품으로만 교환한다.

## 2. 돈의 흐름

광고주가 공급가액 1,000,000원을 집행하는 예시:

| 단계 | 금액 | 비고 |
|---|---|---|
| 광고 공급가액 | 1,000,000원 | 매출 인식 대상 |
| 부가가치세(10%) | 100,000원 | 예수 후 납부 |
| 광고주 결제액 | 1,100,000원 | PG 통해 수취 |
| → 인정 노출에 따라 광고주 과금 원장에서 예산 차감 | | 서빙 예약 → 검증 확정 차감 |
| 개발자 리워드 비용 | 공급가액의 30% (예: 300,000원) | 모델 B 환산치, 정책 상한 내 |
| 쿠폰 공급 원가 + 지급대행 수수료 | 리워드 비용의 일부 | 대량구매 할인으로 체감가 < 액면가 가능 |
| **매출총이익** | 나머지 | 아래 비용 차감 전 |

**차감 비용 항목(리워드 배분율 산정 시 반드시 고려):** VAT, PG 수수료, 쿠폰 구매·발송 수수료, 부정 노출 손실, 환불·미수금, 고객지원, 세무·지급명세 비용, 서버 비용, 광고주 영업 수수료, 리워드 네트워크 수수료.

> "광고 매출의 50%를 사용자에게 준다"는 원본 모델은 한국 비용 구조에서 성립하지 않는다. 아래 모델 B로 대체한다.

## 3. 리워드 배분 — 모델 B (인정 노출당 고정 포인트)

- **인정 노출 1,000회당 300P** (1P = 원화 1원에 상당하되, 현금 등가를 강하게 약속하지 않음 — CLAW-14 참조)
- 광고주 CPM과 **분리 운영**: 광고 단가가 오르내려도 사용자 적립 정책은 독립적으로 유지·조정한다.
- 채택 이유: 사용자가 이해하기 쉽고, 예산 통제가 쉬우며, 광고주 계약과 사용자 리워드를 분리해 경제성을 지킬 수 있다.
- 향후 캠페인별 차등(모델 C)은 P2에서 검토: 예) 클릭 가능 IDE 광고 클릭당 10P, 설문형 완료당 500P.

## 4. 광고주 단가 (직판 기준, 파일럿 v1)

- 고정 CPM **2,000원**(공급가액, VAT 별도) — 개발자 타겟 매체 희소성 반영. 파일럿 데이터로 조정.
- 클릭 과금(IDE 인벤토리, P2): 노출 단가의 50배.
- 최소 집행 금액: 파일럿은 무료/저가, 유료 베타에서 확정.

## 5. 상한 (광고 피로도·어뷰징 방어)

| 항목 | 값(v1) |
|---|---|
| 동일 캠페인 노출 | 사용자당 하루 최대 20회 |
| 동일 광고 재노출 최소 간격 | 10분 |
| 전체 유효 노출 | 사용자당 하루 최대 500회 |
| 일일 적립 상한 | 1,000P |
| 백그라운드 세션 | 유효 노출에서 제외 |

정확한 값은 베타 데이터로 조정한다. 이 수치는 서버 정책 테이블 값이며 클라이언트는 알지 못한다.

## 6. 광고주 예산 차감 시점

```
광고 서빙        → 예산 "예약"(hold)
노출 검증 승인   → 예약분 "확정 차감"
노출 거절/만료   → 예약 "해제"
사후 부정 판정   → 광고주 크레딧 "복원" + 사용자 리워드 "회수"
```

동시 요청에서 예산 초과 집행이 없도록 PostgreSQL 행 잠금/원자적 업데이트로 구현한다(CLAW-17·23).

## 7. 리워드 상태 용어 (화면·DB 공통)

`예상 적립`(클라이언트 미검증) → `검증 중`(서버 심사) → `확정 리워드`(교환 가능) → `교환 대기`(사용자 신청) → `지급 완료`(공급사 발송) / `회수`(부정·취소).

## 8. 미결 사항 (선행 이슈)

- 리워드의 세무상 소득 구분·원천징수 → CLAW-13
- 포인트/상품권 구조의 전자금융거래법 판단 → CLAW-14
- 이 수치들을 담을 정책 테이블 스키마 → CLAW-17(원장)와 함께 구현
