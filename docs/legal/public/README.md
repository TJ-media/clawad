# 법률 공개본 (검토 대기)

이 디렉터리는 운영 서버 `/run/clawad-public-legal/`에 배치할 **공개본 4종**이다.
`docs/legal/` 상위의 초안(`terms-of-service.md`, `privacy-policy-draft.md`)과 달리 실제 게시를 전제로 작성했으나,
**아직 게시 가능 상태가 아니다.** 아래 §1 항목을 채우고 §2 검토를 통과해야 배치한다.

| 파일 | 배치 후 URL | 서버 참조 |
|---|---|---|
| `terms-v1.html` | `/legal/terms-v1.html` | `LEGAL_TERMS_URL` |
| `privacy-v1.html` | `/legal/privacy-v1.html` | `LEGAL_PRIVACY_URL` |
| `privacy-contact.html` | `/legal/privacy-contact.html` | `LEGAL_PRIVACY_CONTACT_URL` |
| `removal-guide.html` | `/legal/removal-guide.html` | `LEGAL_REMOVAL_GUIDE_URL` |
| `_style.css` | `/legal/_style.css` | (공통 스타일) |

파일명은 운영 `.env`의 `LEGAL_*_URL` 값과 일치시킨 것이다. 파일명을 바꾸면 `.env`도 함께 바꿔야 한다.

## 1. 채워야 하는 항목

> **2026-07-20 기준: HTML 4종에 `[미확정:` 마커가 남아 있지 않다.** 아래 §1-1~1-4는 전부 채워졌으며,
> 이력·근거를 남기기 위해 표를 유지한다. 게시 가능 여부는 §2 검토 항목에 달려 있다.

`[미확정: ...]` 마커가 하나라도 남아 있으면 배포 스모크(`scripts/production-smoke.js`)가
실패하도록 설계돼 있으므로, 미완성본이 실수로 게시될 수 없다.

### 1-1. 운영자 식별 정보 — 확정

> **법률 자문 결과 반영(2026-07-20)**: 사업자등록 전 단계이므로 상호·대표자·주소·사업자등록번호·
> 통신판매업 신고번호는 **게시하지 않는다.** 알파는 이용자에게 대가를 받지 않고 유료 광고도 집행하지 않아
> 사업소득이 없으며, 유료 광고 개시 시점에 등록 후 전자상거래법상 사업자 정보를 게시한다(약관 「운영자 정보」에 명시).
>
> 다만 **계약 당사자 식별**과 **개인정보처리자 표시**는 사업자등록과 무관하게 필요하므로 아래는 채워야 한다.

| 항목 | 확정값 | 위치 |
|---|---|---|
| 운영자 표시명 | 김태정 | 약관 제1조·운영자 정보, 방침 서두 |
| 시행일 | 2026-07-14 | 약관 헤더·부칙, 방침 헤더 |

> 시행일은 임의 값이 아니라 **운영 `.env`의 `LEGAL_TERMS_EFFECTIVE_AT`·`LEGAL_PRIVACY_EFFECTIVE_AT`(2026-07-14)와
> DB에 이미 등록된 v1 메타데이터에 맞춘 것이다.** `legal-documents.service.ts`는 같은 버전의 메타데이터 변경을
> 거부하므로, 다른 날짜로 게시하려면 v2로 올리고 `.env`도 함께 바꿔야 한다.

### 1-3. 세무·전자금융거래법 — 마커 없이 확정 문안 처리

CLAW-13·14 서면 답변은 여전히 미확보다. 다만 **답변 없이도 참인 문장으로만 서술**해 마커를 제거했다.

| 위치 | 처리 방식 |
|---|---|
| 방침 제3조 | 보존연수를 숫자로 쓰지 않고 "관계 법령이 정한 기간 동안 보관 후 파기"로 서술. 법령 개정 시 방침을 개정해 고지한다는 문장 추가 |
| 약관 제8조 안내상자 | 리워드의 전자금융거래법상 지위·소득 구분을 "검토를 거쳐 확정되는 대로 반영하여 고지"로 서술 |

> **세율·과세 기준·규제 비해당 판단을 확정 사실로 기술하지 않는다는 원칙은 유지된다.**
> 위 문장을 단정형(예: "기타소득에 해당합니다", "보존기간은 5년입니다")으로 바꾸려면
> CLAW-13·14 서면 답변이 선행돼야 한다.
>
> 요청받은 "세무 기록 2년"은 기재하지 않았다. 사업자등록 후에는 국세기본법상 장부·증거서류 보존의무가
> 적용되어 2년보다 길 수 있고, 문서에 짧은 연수를 단정해 두면 실제 의무와 어긋날 위험이 있다.
> 위 서술 방식은 법령이 정하는 기간을 그대로 따르므로 연수가 무엇이든 항상 정확하다.

## 3. 배치 방법

운영 서버 접속 명령과 인스턴스 식별자는 공개 저장소에 두지 않는다. 접속 명령은 terraform 출력에서 얻고, terraform state가 없으면 Jira CLAW-60의 운영 인수인계 코멘트를 따른다.

```bash
terraform -chdir=deploy/terraform/aws output ssm_command
```

운영 서버에 접속한 뒤:

```bash
sudo mkdir -p /run/clawad-public-legal
# 5개 파일을 /run/clawad-public-legal/ 에 배치 (파일명 유지)
sudo chmod 0644 /run/clawad-public-legal/*
```

배치 후 검증:

```bash
npm run infra:prod:smoke   # 로컬에서 실행. 미확정 마커가 남아 있으면 실패한다
```

수동 확인:

```
https://clawad.whatsup.house/legal/terms-v1.html
https://clawad.whatsup.house/legal/privacy-v1.html
https://clawad.whatsup.house/legal/privacy-contact.html
https://clawad.whatsup.house/legal/removal-guide.html
```

## 4. 작성 시 반영한 사실 (검토자 참고)

문안은 다음 구현 사실에 맞춰 작성했다. 코드가 바뀌면 문서도 함께 고쳐야 한다.

- 로그인은 소셜 3사 전용, **이메일·이름·프로필 미요청**(`google.provider.ts` scope=openid)
- 클라이언트 전송 필드 8개로 한정 (`docs/legal/privacy-design.md` §1)
- 상품 수령 이메일은 **교환 시점에 입력·동의**, 발송/취소/실패 종결·탈퇴 시 즉시 파기 (CLAW-74)
- 접속 IP는 이벤트 원장에 저장하지 않음
- 기기 최대 3대, 동시 노출은 1건만 인정하되 **제재 아님** (CLAW-19 정책)
- 자동 판정만으로 제재하지 않고 지급 보류 후 운영자 검토 우선
- 인프라는 AWS 서울 리전(ap-northeast-2) → **국외 이전 "해당 없음"으로 확정 기술**
- 알파 단계 쿠폰은 운영자 직접 발송 → **처리위탁 "없음"으로 확정 기술**
- 데이터 내보내기 `GET /v1/me/export`, 탈퇴 `DELETE /v1/me` 구현됨 (CLAW-28)
- 일시중지 `npm run clawad:pause`, 제거 `npm run clawad:uninstall`

### 초안 대비 달라진 판단

- **광고주 약관(초안 §B)은 공개본에서 제외했다.** 알파는 HOUSE·TEST 캠페인만 운영해 외부 광고주가 없고,
  초안의 미확정 항목 2건(최소 집행 금액, 리포트 확정 시점)이 모두 §B에 속해 공개를 지연시킨다.
  광고주 온보딩 시점에 별도 문서로 게시한다.
- 초안에 없던 **자동화된 결정에 관한 조항**(방침 제10조)을 추가했다. 무효 트래픽 자동 검증이
  리워드 산정에 영향을 주므로 개인정보보호법 §37조의2 대응이 필요하다고 판단했다. 변호사 검토 항목.
