# 리워드 구조의 전자금융거래법(EFAA) 검토 (CLAW-14)

> 상태: **법률 검토 대기**. 이 문서는 자문 요청용 정리이며 확정 의견이 아니다.
> 기존 가정 "잔액을 짧게 유지하면 선불전자지급수단 규제를 회피한다"는 **폐기**한다. 명칭을 "포인트"로 한다고 규제 대상에서 벗어나지 않는다.

## 1. 판단 기준 (자문 시 확인 항목)

전자금융거래법상 선불전자지급수단·전자금융업 해당 여부는 아래를 종합 판단한다. 각 항목에서 클로애드의 목표 설계(규제에서 멀어지는 방향)를 병기한다.

| 기준 | 규제 접근 요소 | 클로애드 목표 설계 |
|---|---|---|
| 발행 주체 | 자체 범용 지급수단 발행 | 클로애드는 발행하지 않음. 외부 지급대행사가 쿠폰 발행 |
| 금전적 가치 청구권 | 사용자가 현금 청구 가능 | 현금 청구 불가. 지정 상품 교환만 |
| 제3자 가맹점 사용 | 다수 가맹점 직접 결제 | 직접 결제 불가. 클로애드가 상품 구매 후 전달 |
| 양도 가능성 | 사용자 간 이전 가능 | 양도·이전 불가 |
| 환금성 | 현금·타 지급수단 환급 | 환급 불가 |
| 충전 가능성 | 사용자가 충전·구매 | 충전·구매 불가. 광고 참여로만 적립 |
| 발행 잔액·이용 규모 | 대규모 미상환 잔액 | 적립-교환 회전으로 잔액 최소화(단, 이것만으로 회피 불가) |
| 클로애드의 역할 | 지급수단 발행자/중개자 | 광고 매체 사업자. 지급은 대행사 위탁 |

## 2. 규제에서 멀어지는 리워드 설계 원칙 (확정)

> **클로애드 리워드는 광고 참여에 따라 부여되는 비구매형·비양도형 보상이다. 클로애드가 지정한 상품으로만 교환할 수 있으며, 충전·양도·사용자 간 이전·현금 환급을 지원하지 않는다.**

금지 기능(구현 자체를 하지 않음):
- "1P = 현금 1원, 언제든 출금" 강한 약속 표시
- 포인트 충전·구매
- 사용자 간 포인트 이전·선물
- 다수 제3자 가맹점 직접 결제

## 3. 설계 강제 요건 (DB·화면·약관 3중 강제)

이 원칙은 문구뿐 아니라 시스템으로 강제한다:

- **DB 제약**: 리워드 원장에 사용자 간 이전(transfer) 트랜잭션 타입 부재. 잔액 → 현금 전환 경로 부재. 교환 대상은 상품 카탈로그 화이트리스트로만.
- **화면 문구**: 수익은 "예상/확정 리워드(P)"로 표기, "현금 출금" 문구 미사용(P2 현금 도입 시 별도 검토).
- **약관(CLAW-16)**: 리워드의 비구매·비양도·교환 전용 성격 명시.

## 4. 지급대행 구조 (발행자 아님 논증)

- 사용자가 확정 리워드로 교환 신청 → 클로애드가 **지급대행사로부터 상품(쿠폰)을 구매** → 대행사가 사용자에게 발송.
- 클로애드는 지급수단을 발행·운영하지 않고, 상품 구매·전달의 중개/구매자 지위에 선다.
- 이 구조가 전금법상 안전한지 자문으로 확인 (특히 대행사 계약서의 책임 배분).
